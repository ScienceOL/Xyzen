# Configuration

Xyzen is configured entirely through environment variables, all prefixed with `XYZEN_`. {{ className: 'lead' }}

Set these in your `.env` file (for Docker Compose) or ConfigMap (for Kubernetes).

## Core Settings

| Variable | Description | Default |
|---|---|---|
| `XYZEN_TITLE` | Application title | `Xyzen` |
| `XYZEN_DESCRIPTION` | Application description | — |
| `XYZEN_SECRET` | Secret key for token signing | **Required** |
| `XYZEN_ENV` | Environment (`development`, `production`) | `development` |
| `XYZEN_DEBUG` | Enable debug mode | `false` |
| `XYZEN_HOST` | Server bind host | `0.0.0.0` |
| `XYZEN_PORT` | Server bind port | `48196` |
| `XYZEN_LOG_LEVEL` | Logging level | `info` |

## Authentication

Xyzen supports OAuth/OIDC authentication through external providers:

| Variable | Description | Default |
|---|---|---|
| `XYZEN_AUTH_PROVIDER` | Auth provider (`casdoor`, `bohrium`) | — |
| `XYZEN_AUTH_ISSUER` | OAuth issuer URL | — |
| `XYZEN_AUTH_AUDIENCE` | OAuth client ID / audience | — |
| `XYZEN_AUTH_JWKS_URI` | JWKS endpoint for token verification | — |
| `XYZEN_AUTH_ALGORITHM` | JWT signing algorithm | `RS256` |

When no auth provider is configured, Xyzen runs in open access mode.

## Database & Redis

| Variable | Description | Default |
|---|---|---|
| `XYZEN_DATABASE_URL` | PostgreSQL connection string | `postgresql+asyncpg://postgres:postgres@postgresql:5432/postgres` |
| `XYZEN_REDIS_URL` | Redis connection string | `redis://redis:6379/0` |

### PostgreSQL Notes

- Uses async driver (`asyncpg`) for non-blocking queries
- Migrations run via Alembic (`just migrate-up`)
- SQLite is supported for development but not recommended for production

## LLM Providers

Configure one or more LLM providers. Agents select models from the configured providers.

### OpenAI

| Variable | Description |
|---|---|
| `XYZEN_OPENAI_API_KEY` | OpenAI API key |
| `XYZEN_OPENAI_API_BASE` | Custom base URL (for compatible APIs) |
| `XYZEN_OPENAI_MODEL` | Default model |

### AWS Bedrock

| Variable | Description |
|---|---|
| `XYZEN_BEDROCK_REGION` | AWS region |
| `XYZEN_BEDROCK_ACCESS_KEY` | AWS access key |
| `XYZEN_BEDROCK_SECRET_KEY` | AWS secret key |

### Google Vertex AI

| Variable | Description |
|---|---|
| `XYZEN_GOOGLE_PROJECT_ID` | GCP project ID |
| `XYZEN_GOOGLE_REGION` | GCP region |
| `XYZEN_GOOGLE_CREDENTIALS` | Path to service account JSON |

### Azure OpenAI

| Variable | Description |
|---|---|
| `XYZEN_AZURE_OPENAI_API_KEY` | Azure API key |
| `XYZEN_AZURE_OPENAI_ENDPOINT` | Azure endpoint URL |
| `XYZEN_AZURE_OPENAI_DEPLOYMENT` | Deployment name |

## Object Storage

Files are stored in S3-compatible object storage:

| Variable | Description | Default |
|---|---|---|
| `XYZEN_S3_ENDPOINT` | S3 endpoint URL | `http://minio:9000` |
| `XYZEN_S3_ACCESS_KEY` | Access key | — |
| `XYZEN_S3_SECRET_KEY` | Secret key | — |
| `XYZEN_S3_BUCKET` | Bucket name | `xyzen` |
| `XYZEN_S3_REGION` | Region | `us-east-1` |

## Sandbox

| Variable | Description | Default |
|---|---|---|
| `XYZEN_SANDBOX_BACKEND` | Sandbox backend (`runner`, `daytona`) | `runner` |
| `XYZEN_SANDBOX_TIMEOUT` | Execution timeout in seconds | `300` |

### Daytona Backend

| Variable | Description |
|---|---|
| `XYZEN_DAYTONA_API_URL` | Daytona API endpoint |
| `XYZEN_DAYTONA_API_KEY` | Daytona API key |

## Search

| Variable | Description | Default |
|---|---|---|
| `XYZEN_SEARXNG_URL` | SearXNG instance URL | `http://searxng:8080` |
