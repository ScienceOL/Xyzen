# 设计理念

Xyzen 由一个信念驱动：好奇心不应被资源所限，研究不应被规模所困。 {{ className: 'lead' }}

## 从"能做"到"会想"

最初，我们埋头写一行行代码，试图解决一个看似简单却极具挑战的问题：如何让大模型真正"**动手**"——调用搜索、生成代码、访问数据库、操作文件……甚至接入外部 API。计算本身变成了 Agent 的"双手"。

然而，在迈出"手"的坚实一步后，我们陷入了新的困惑：**这样够吗？** 科学家需要的不是一个听话的流水线工人，而是能理解模糊意图、主动提出方案的思考伙伴。

**从"能做"到"会想"。**

于是，我们重构了产品逻辑。引入图模编排和 DSL，让科研者能半自然地设计自己的思维路径；搭建多 Agent 协作架构，让一次深度研究能像人类科研那样——拆解、假设、验证、归纳。（这也是 Xyzen 原本只存在于记忆当中的名字 *Zen* 的由来。）

## Agent 即资产

正因如此，我们提出了"**Agent 即资产**"的理念。

每一个精心训练的 Agent，都是研究者隐性知识的结晶，是可以被传承、复用和演进的方法论的具体形态。它不是用完即弃的工具，而是智力资本。

当研究者花费大量时间打磨 Agent 的系统提示词、附加合适的技能、调整工具组合时，他们正在将领域专业知识编码为一个活的、可执行的制品。这个制品可以与合作者共享，发布到[市场](/marketplace)，或者被下一代研究者复刻和改进。

## 科学思维的数字孪生

想象这样的未来：你的 Agent 将不再只是工具，而是你科学思维的"**数字孪生**"——它理解你的逻辑，记录你的学术品味，在你休息时继续探索你的假说，在你回来时带着全新的科研灵感和准备好的初稿。

这就是我们正在构建的愿景。通过[记忆](/memory)，Agent 跨对话积累关于你的知识；通过[技能](/skills)，它学习你的方法论；通过[知识库](/knowledge)，它内化你的数据。随着时间推移，Agent 逐渐成为你智识自我的延伸。

## 科学民主化

这就是我们想通过 Xyzen 传递的信念：**让一个人也能拥有一整个自驱动的实验室，一个小团队也能攻克曾经只有大机构才能触及的问题。**

这不仅是科技的进步，更是**科学民主化**（Democratization of Science）的实现——让好奇心不再受限于资源，让研究不再被规模绑架。

让科研有温度，真正地去科学探索。孩子般的好奇心，就是一切的驱动力——它驱动了每一行代码、每一次重构、每一次对 Agent 更深层次理解的追求。让 Xyzen 成为那个桥梁——在浩瀚未知面前，帮你以一人之力，叩问星辰。
