# API Reference

All Xyzen functionality is accessible via REST API. The API uses JSON for request and response bodies, Bearer token authentication, and SSE for real-time streaming. {{ className: 'lead' }}

## Base URL

All API endpoints are prefixed with `/xyzen/api/v1/`:

```
https://your-domain.com/xyzen/api/v1/
```

## Authentication

Most endpoints require a Bearer token in the `Authorization` header:

```bash
curl -H "Authorization: Bearer {access_token}" \
  https://your-domain.com/xyzen/api/v1/agents
```

### Auth Endpoints

| Method | Endpoint | Description |
|---|---|---|
| `GET` | `/auth/status` | Check if auth is configured |
| `GET` | `/auth/config` | Get OAuth provider configuration |
| `POST` | `/auth/login/casdoor` | Exchange OAuth code for token |
| `POST` | `/auth/validate` | Validate token, get user info |

## Chat

Chat uses REST endpoints for sending messages and SSE for receiving responses:

### Sending Messages

| Method | Endpoint | Description |
|---|---|---|
| `POST` | `/topics/{id}/messages` | Send a message |
| `POST` | `/topics/{id}/abort` | Abort current generation |
| `POST` | `/topics/{id}/regenerate` | Regenerate last response |
| `POST` | `/topics/{id}/question-response` | Respond to agent question |

### Receiving Events

```bash
# Connect to SSE stream for a topic
curl -N -H "Authorization: Bearer {token}" \
  -H "Accept: text/event-stream" \
  https://your-domain.com/xyzen/api/v1/topics/{id}/events
```

The SSE endpoint supports `Last-Event-ID` for reconnection. See [Streaming Events](#streaming-events) for event types.

## Streaming Events

Events are delivered via SSE with the following format:

```
id: 1234567890-0
event: streaming_chunk
data: {"content": "Hello", "node_id": "response"}
```

### Event Types

| Event | Data Fields | Description |
|---|---|---|
| `loading` | `message` | Agent is starting up |
| `agent_start` | `agent_type` | Agent execution began |
| `node_start` | `node_id`, `node_type` | A new phase started |
| `streaming_start` | — | Streaming content is beginning |
| `streaming_chunk` | `content`, `node_id` | A chunk of text content |
| `thinking_chunk` | `content` | A chunk of reasoning content |
| `node_end` | `node_id` | Phase completed |
| `streaming_end` | — | Streaming complete |
| `agent_end` | `status` | Agent execution finished |
| `message_saved` | `message_id` | Message persisted to database |

### Reconnection

If the SSE connection drops, reconnect with the `Last-Event-ID` header:

```bash
curl -N -H "Last-Event-ID: 1234567890-0" \
  -H "Authorization: Bearer {token}" \
  https://your-domain.com/xyzen/api/v1/topics/{id}/events
```

The server replays all events after the given ID, then switches to live streaming. This guarantees zero data loss.
